# 🏈 NFL Oracle — AI Game Predictor

A full-stack NFL game prediction app using **scikit-learn** (Gradient Boosting),
**nfl_data_py** for real play-by-play data, and **FanDuel odds** via The Odds API.

---

## Features

- **Real data** — Play-by-play from `nfl_data_py` (nflverse), covering the last 4 seasons
- **Auto-updating seasons** — Seasons window shifts automatically each September
- **ML model** — `GradientBoostingClassifier` (scikit-learn) trained on 25 rolling features
- **Rolling 5-game windows** — Each team's last 5 games compute features (EPA, points, turnovers, etc.)
- **Mid-season trade support** — Patch any team's rolling stats via `/api/update-profile` without retraining
- **Vegas odds** — Live from FanDuel via The Odds API (free API key), or estimated from rolling stats
- **Team filter** — Filter by team, conference (AFC/NFC), or prediction confidence
- **Custom matchup predictor** — Pick any two teams and get an instant prediction

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Train the model

This downloads nfl_data_py data (~2–5 min first run), builds features, trains, and saves the model:

```bash
python -m scripts.train
```

You should see output like:
```
CV Accuracy: 0.5832  (+/- 0.0241)
Model saved  → app/models/model.pkl
Profiles saved → app/models/team_profiles.pkl
```

### 3. (Optional) Add FanDuel odds

Sign up free at https://the-odds-api.com (500 requests/month free).
Set your API key as an environment variable:

```bash
export ODDS_API_KEY=your_key_here
```

Without a key, the app estimates spreads/moneylines from rolling team stats.

### 4. Start the server

```bash
python run.py
```

Open **http://localhost:5000** in your browser.

## API Endpoints

| Method | Endpoint              | Description                              |
|--------|-----------------------|------------------------------------------|
| GET    | `/api/health`         | Server + model status                    |
| GET    | `/api/model-info`     | CV accuracy, features, importances       |
| GET    | `/api/teams`          | All teams + rolling profiles             |
| GET    | `/api/upcoming`       | Upcoming week predictions + Vegas odds   |
| GET    | `/api/upcoming?team=KC` | Filter to a specific team              |
| POST   | `/api/predict`        | Custom matchup prediction                |
| POST   | `/api/update-profile` | Patch team stats (mid-season trades)     |

### POST /api/predict
```json
{ "home_team": "KC", "away_team": "BUF" }
```

### POST /api/update-profile
```json
{
  "team": "NE",
  "stats": {
    "roll_offensive_epa": 0.14,
    "roll_pass_epa": 0.20
  }
}
```

Valid stat keys: `roll_offensive_epa`, `roll_defensive_epa_allowed`, `roll_pass_epa`,
`roll_rush_epa`, `roll_points_scored`, `roll_points_allowed`, `roll_turnovers`, `roll_won`

## Model Features (25 total)

Each feature is a rolling 5-game average:

**Home team (8):** offensive EPA/play, defensive EPA allowed, pass EPA, rush EPA,
points scored, points allowed, turnovers, win rate

**Away team (8):** same as above

**Differentials (8):** home minus away for each stat

**Home advantage (1):** always 1 during training

---

## Retraining

The model auto-uses the 4 most recent seasons. Re-run training at the start of each season:

```bash
python -m scripts.train
```

---

## Notes

- EPA (Expected Points Added) is the primary signal — it's a stronger predictor than raw yards/points
- Home field advantage is baked in as a feature (always set to 1 for home team)
- Vegas odds are for comparison only — the ML model is independent of them
- For entertainment purposes only. Please gamble responsibly.
